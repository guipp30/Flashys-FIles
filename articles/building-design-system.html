Building a Design Token System from Scratch

By Flashy

Published April 15, 2026

Design tokens are the atomic elements of your product's visual identity. Here is the step-by-step framework to transition from hard-coded hex values to a bulletproof token architecture that engineers will actually love using.

1. Naming Conventions

The biggest mistake teams make is jumping straight into semantic tokens without defining the global scale first. Start with a neutral palette and establish a clear naming framework. We recommend the Category-Property-Concept-State format. 

/* Global tokens */
--color-blue-lower: #1B4B8;
--global-color-blue-800: #282948;
/* ... */

2. Structuring the Figma File

Keep your tokens isolated from your component library. Create a dedicated 'Foundations' file that publishes exclusively styles and variables.

3. Handling Dark Mode

Dark mode should never require duplicating components. By mapping light and dark themes to the same semantic token names, the switch happens automatically at the variable level.

"A token system isn't finished when it works in Figma. It's finished when a developer can build a new feature without ever sampling a color."

Take the time to align your variable collections with your codebase's theme provider. The initial setup takes hours, but it saves weeks of back-and-forth pixel pushing.
